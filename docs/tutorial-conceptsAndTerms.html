    Validators

    Think of validators as the atoms of this module - all paramter valiation relies on testing values agsinst one or more validators.

    A validator is simply a function. Its input is the value under test and some configuration information, or validator options, and its output will be either one or more error messages, or, undefined.

    Constraints

    If you think of validators as atoms, then think of constraints as molecules - collections of validators with associated validator options.

    Constaints are represented as JavaScript object literals where each key is the name of a validator, and each value the associated validator option data.

    The validator option data does not have to be complex, it all depends on the specific validator. For example, the presence validator simply expects a boolean for its option data.
